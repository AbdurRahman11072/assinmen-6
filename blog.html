Question 1: Discuss the scope of var, let, and const

In JavaScript, var, let, and const are used to declare variables, each with its own scope and behavior.
var:
Variables declared with var are function-scoped, meaning they are accessible throughout the entire function in which they are defined.
They are hoisted to the top of their function or global scope, allowing them to be used before declaration with an initial value of undefined.
var variables can be reassigned multiple times within their scope.
let:
Variables declared with let are block-scoped, meaning they are confined to the block (enclosed within curly braces) in which they are defined. They are not accessible outside that block.
They are hoisted but not initialized, resulting in a ReferenceError if used before declaration.
let variables can be reassigned within their block scope.
const:
Like let, variables declared with const are block-scoped.
They are hoisted but not initialized, leading to a ReferenceError if used before declaration.
const variables are constant and cannot be reassigned after initialization, making them suitable for declaring constants or values that should not change.
In summary, var has function scope, let and const have block scope, and const additionally enforces immutability after declaration. It is generally recommended to use let and const for better scoping and predictability, while var is considered outdated in modern JavaScript due to its potential for hoisting-related bugs.

Question 2: Tell us the use cases of null and undefined

'null' and 'undefined' are two distinct values in JavaScript used to represent missing or non-existent values, but they have slightly different use cases:
'undefined':
Typically indicates that a variable has been declared but has not been assigned a value.
It also serves as the default value for function parameters that are not provided.
Often arises when trying to access object properties that do not exist.
Can be used explicitly to clear the value of a variable or object property.
Indicates a lack of a return value for a function that does not explicitly return anything.
It can cause unexpected behavior in code if not handled properly.
'null':
Represents an intentional absence of any object value or no value at all.
It is often used to signify that a variable or object property has no meaningful value or is explicitly empty.
Can be used as a placeholder for an object property that will later hold an object reference.
Provides a clear and intentional way to represent the absence of an object, whereas undefined is more likely to indicate a mistake or unintentional absence.
In summary, undefined is often unintentional and indicates the absence of assignment, while null is a deliberate choice to represent a lack of value or emptiness in a variable or object property. Properly handling these values is essential to avoid bugs and ensure predictable behavior in your JavaScript code.

Question 3: What do you mean by REST API?

A REST API (Representational State Transfer Application Programming Interface) is a web-based architecture for communication between different software systems. It uses standard HTTP methods (such as GET, POST, PUT, DELETE) to allow clients to interact with and manipulate resources (data) on a server, typically using a stateless, client-server approach.
